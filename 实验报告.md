## 三、实验体会

### 1. 项目选题与整体设计

我们小组选择开发一个以 Supercell 手游《荒野乱斗》（Brawl Stars）为主题的资料展示站，定位为“图鉴 + 赛事”站点。项目主要面向中文用户，核心功能包括：

- **英雄图鉴**：英雄列表、分类筛选、英雄详情页（技能、星辉、妙具、皮肤、背景故事等）；
- **对战地图**：地图列表、按模式/实时天梯池筛选、地图收藏；
- **游戏模式**：模式卡片、实时轮换倒计时、模式详情页；
- **赛事中心**：赛事列表、战队/选手档案、赛程日历、数据统计、淘汰赛进程树；
- **用户系统**：注册登录、收藏英雄/地图/赛事、浏览历史、个人分享卡片。

在技术选型上，我们采用 **Next.js 16 App Router + React 19 + TypeScript + Tailwind CSS v4** 作为前端框架，利用其服务端渲染、静态生成和 API Routes 能力；后端使用 Next.js API Routes + **Prisma 7 + PostgreSQL（Neon）** 处理用户认证与持久化数据；认证方案采用 **Auth.js v5 beta** 的 Credentials Provider + JWT Strategy。

### 2. 开发过程中遇到的问题及解决方法

#### （1）赛事数据来源复杂、格式不统一

赛事中心需要展示全球范围内的月度决赛、LCQ、全球总决赛、Brawl Cup 等数据。这些赛事信息分散在 Liquipedia 等多个来源，且页面结构差异大。我们编写了 Python 爬虫脚本 `scripts/fetch_liquipedia_esports.py`，对不同赛制的页面进行解析，并将抓取到的 JSON 数据合并到 `lib/data/liquipediaTournaments.json` 中。脚本采用按赛事 ID 去重的策略，重新运行时不会覆盖已有数据，保证了历史赛事的完整性。

#### （2）数据库从 SQLite 迁移到 PostgreSQL

项目最初使用 SQLite 进行本地开发，但在准备部署到 Vercel 时发现 SQLite 不适合无服务器环境。我们将数据库迁移到 **Neon PostgreSQL**，并更新 Prisma schema、adapter 配置和构建脚本（`prisma generate && next build`）。迁移过程中最大的注意点是连接字符串和环境变量的管理，最终通过 `.env.local` 和 Vercel 环境变量完成配置。

#### （3）认证部署后出现“Server configuration”错误

本地开发正常，部署到 Vercel 后 Auth.js 报配置错误。原因是 Auth.js v5 beta 在某些环境下不会自动读取 `AUTH_SECRET`。修复方法是在 `auth.ts` 中显式设置 `secret: process.env.AUTH_SECRET`，问题得以解决。

#### （4）2025 赛事在 UI 中不可见

完成 2026 北美/南美赛事数据补充后，发现 2025 年的 LCQ、Global Finals、Brawl Cup 在页面上“消失”。排查后发现数据本身仍在 JSON 中（共 178 场赛事），只是前端月份筛选器缺少 `2025-05` 等月份选项。我们在 `/events/page.tsx` 和 `/events/schedule/page.tsx` 中补齐了月份，并新增了年份筛选，最终恢复了全部历史赛事的可见性。

#### （5）Brawl Cup 淘汰赛树状图展示不合理

2025 Brawl Cup 的赛制是“小组双败 + 季后赛单败”，但通用淘汰赛组件只能按八强赛/半决赛/总决赛扁平展示，无法表达小组赛阶段。为此我们专门编写了 `BrawlCup2025Bracket` 组件，区分 Group A、Group B 和 Playoffs，用多列卡片形式还原真实对阵结构。

#### （6）英雄推荐和地图推荐过于单一

“猜你喜欢”最初只按用户收藏最多的单一模式/职位排序推荐，导致即使用户收藏了多个定位，推荐结果也全部是同一种类。我们改为**按收藏分布做轮询推荐**：把每种模式/职位按权重排序，依次从各类别中取候选，最终让多个偏好都能在推荐列表中得到展示。

#### （7）英雄品质显示异常、贴图错位

部分英雄的品质名称仍显示英文（Rare）、颜色值存在 `#fff11ev` 这类非法 hex，且沙迪等英雄的本地 Sprite 贴图与英雄 ID 不匹配（沙迪显示成了芽芽）。我们新增 `lib/rarity.ts` 统一映射中英文品质名和标准颜色；同时将 `scripts/fetch-heroes.mjs` 中的图片解析改为统一使用 Brawlify CDN 模型图，避免本地 Sprite 编号错乱的问题。

#### （8）Git 推送受网络影响

在某几次提交时，本地到 GitHub 的 HTTPS 连接出现 `SSL_ERROR_SYSCALL` 错误。通过切换网络/开启代理后重新 `git push origin main` 解决。

### 3. 最终实验结果

目前项目已完成并部署到 Vercel，访问地址为：

> https://brawlstars-showcase.vercel.app/

主要完成的功能和指标如下：

- 英雄数据：104 位英雄，包含属性、技能、星辉、妙具、装备、极限充能、皮肤等；
- 地图数据：通过 Supercell 官方 API 代理获取，支持实时轮换和天梯池筛选；
- 赛事数据：178 场赛事、覆盖 2025–2026 多赛区月度决赛、LCQ、全球总决赛、Brawl Cup；
- 用户系统：注册登录、地图/英雄/赛事收藏、浏览历史、个人分享卡片；
- 代码质量：`npm run lint` 0 错误，`npm run build` 构建成功；
- 响应式布局：适配桌面端和移动端，UI 使用 Tailwind CSS v4 完成。
