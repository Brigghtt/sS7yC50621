一、基础结构标签（网页骨架）
    <html>
    意义：整个网页的根标签，所有内容都包在里面
    用法：一个页面只能有一个
    <head>
    意义：网页头部，放配置、标题、引入样式，页面上看不见
    用法：放标题、图标、CSS、meta 设置
    <body>
    意义：网页主体，所有能看到的内容都放这里
    用法：文字、图片、导航、按钮全写在 body 里
    <title>
    意义：浏览器标签页的标题文字
    用法：<title>荒野乱斗</title>
二、文本基础标签（写文字用）
    <h1> ~ <h6> 标题标签
    意义：1 最大、6 最小，页面标题 / 小标题
    用法：
    <h1> 页面主标题（只能一个）
    <h2> 板块标题
    <h3> 小副标题
    <p> 段落标签
    意义：包裹一段正文文字，自带上下间距
    用法：<p>这是一段介绍文字</p>
    <span> 行内小容器
    意义：只用来改局部文字样式，不换行、无默认样式
    用法：给一小段文字单独改颜色、大小
    <b> / <strong> 加粗
    <b>：单纯文字加粗
    <strong>：加粗 + 强调语义（搜索引擎更认可）
    <i> / <em> 斜体
    <i>：单纯斜体
    <em>：斜体 + 强调语义
    <br> 换行
    意义：强制文字换行，单标签不用闭合
    <hr> 分割线
    意义：页面水平分割线
三、布局容器标签（做页面结构核心）
    <div> 块级大容器
    意义：万能盒子，用来装一堆内容、做布局、分板块
    特点：默认独占一行
    用法：导航栏、卡片、页脚全部用 div 包
    <nav> 导航栏
    意义：专门放导航菜单的语义标签
    用法：网站顶部导航、侧边导航都用它
    <header> 头部区域
    意义：页面 / 板块的头部，放 logo、导航
    <main> 主体内容
    意义：页面主要内容区域，一个页面只一个
    <footer> 页脚
    意义：网页最底部，放版权、联系方式
    <section> 板块区域
    意义：把页面分成一个个独立板块，比 div 更有语义
四、链接 & 跳转标签
    <a> 超链接
    属性 href="地址"：要跳转的路径
    属性 target="_blank"：新标签页打开
    用法：
    html
    预览
    <a href="/heroes">英雄图鉴</a>
    Next.js 专属 <Link>
    意义：替代 <a>，无刷新跳转页面
    必须配 href 属性
    只在 Next.js 项目里用
五、图片 & 媒体标签
    <img> 图片
    属性 src="图片地址"：图片路径
    属性 alt="描述"：图片加载失败时显示文字
    单标签，不用闭合
    <video> 视频
    意义：嵌入播放视频
    常用属性 controls 显示播放控件
六、列表标签
    <ul> + <li> 无序列表（圆点）
    <ul>：列表大盒子
    <li>：每一项列表内容
    做导航菜单、选项列表必用
    <ol> + <li> 有序列表（数字 123）
    自动带数字序号
七、表单标签（输入框、按钮）
    <input> 输入框
    type="text" 文字输入
    type="password" 密码框
    type="button" 普通按钮
    type="submit" 提交按钮
    <button> 按钮
    意义：可点击按钮，自带按钮样式，可放文字 / 图标
    <textarea> 多行文本框
    意义：可以输入多行文字
    <select> + <option> 下拉选择框
八、行内元素 vs 块级元素（新手必记）
    块级元素（独占一行）
    div p h1~h6 nav header section
    可以设宽高、上下 margin
    行内元素（不换行，并排显示）
    span a img b i
    不能随便设宽高，只占内容本身大小
    九、你写项目最常用标签速记
    布局：div nav header footer文字：p span h1-h3跳转：a / Next.js Link图片：img列表：ul li按钮：button


    [前缀]-[方向]-[尺寸值]
    1. 前缀（决定「是什么类型的尺寸」）
表格
前缀	英文全称	本质含义	作用
w	width	宽度	设置元素的宽度
h	height	高度	设置元素的高度
max-w	maximum width	最大宽度	设置元素的宽度上限（自适应不超过这个值）
max-h	maximum height	最大高度	设置元素的高度上限
min-w	minimum width	最小宽度	设置元素的宽度下限（再小也不会缩到比它还窄）
min-h	minimum height	最小高度	设置元素的高度下限
p	padding	内边距	元素内容和边框之间的距离（内部空白）
m	margin	外边距	元素和其他元素之间的距离（外部空白）

2. 方向（决定「作用在哪个方向 / 边」）
方向只在 p 和 m 里用，w/h 是整体宽高，不用方向。
表格
方向	英文全称	本质含义	作用
（无方向）	无	全方向	上下左右都生效（如 p-4）
x	x-axis	X 轴（水平方向）	左右两边生效（如 px-4）
y	y-axis	Y 轴（垂直方向）	上下两边生效（如 py-4）
t	top	顶部	只作用在顶部（如 pt-4）
r	right	右侧	只作用在右侧（如 pr-4）
b	bottom	底部	只作用在底部（如 pb-4）
l	left	左侧	只作用在左侧（如 pl-4）

二、尺寸值系列 1：数字系列（1 ~ 96）
1. 命名来源 & 原理
Tailwind 有一套固定的数字 → 像素换算规则，本质是基于 0.25rem 为基础单位：
基础单位：1 = 0.25rem
换算公式：n * 0.25rem（rem 是相对于根字体大小的单位，默认根字体是 16px，所以 1rem = 16px）
